# Тест-план: Finik — Приложение для управления личными финансами

**Версия:** 1.1  
**Дата:** 02.03.2026  
**Платформы:** iOS 18.6+, macOS 26.0+, visionOS 26.1+  
**Тип тестирования:** Функциональное, регрессионное, UI/UX, граничные значения, кросс-платформенное  

---

## 1. Введение

### 1.1 Цель документа

Данный тест-план описывает стратегию, объём и подходы к тестированию приложения Finik. Документ предназначен для QA-специалистов и охватывает все функциональные модули приложения.

### 1.2 Объект тестирования

Finik — мобильное и десктопное приложение для управления личными финансами. Основной рынок — Беларусь (валюта по умолчанию BYN). Приложение работает на iOS, macOS и visionOS без внешних зависимостей (только Apple frameworks).

### 1.3 Тестовое окружение

| Параметр | Значение |
|---|---|
| iOS устройства | iPhone 15 Pro, iPhone SE (3rd gen), iPad Pro |
| macOS | MacBook с macOS 26.0+ |
| Xcode | 26.1+ |
| Язык интерфейса | Русский |
| Валюта по умолчанию | BYN |

### 1.4 Инструменты

- Xcode Simulator (iOS, macOS)
- Swift Testing (`@Suite`, `@Test`, `#expect`) для unit/integration тестов
- FinikUITests для UI smoke-сценариев
- TestFlight для бета-тестирования
- Ручное тестирование на реальных устройствах

---

## 2. Стратегия тестирования

### 2.1 Приоритеты

| Приоритет | Модули |
|---|---|
| P1 (Критический) | Транзакции (вкл. переводы), Счета, Безопасность/блокировка, Бюджеты |
| P2 (Высокий) | Онбординг, Планирование (бюджеты/платежи), Импорт выписок, Локальные уведомления |
| P3 (Средний) | Дашборд/Аналитика/AI-инсайты, Курсы валют, Экспорт/импорт архива (.finik) |
| P4 (Низкий) | Банковские калькуляторы, Оптимизация долгов, Виджеты и расширенные UX-сценарии |

### 2.2 Типы тестирования

- Smoke-тестирование (базовая работоспособность)
- Функциональное тестирование (по модулям)
- Граничные значения (boundary value analysis)
- Негативное тестирование (некорректные данные)
- Регрессионное тестирование (после изменений)
- UI/UX тестирование (соответствие дизайну)
- Тестирование производительности (большие объёмы данных)
- Тестирование доступности (VoiceOver, Dynamic Type)
- Кросс-платформенное тестирование (iOS vs macOS)

## 26. Известные ограничения и особенности тестирования

- Сканирование чеков (ReceiptScanWizard) доступно только на реальном iOS-устройстве с камерой.
- Apple Foundation Models AI доступен только на iOS 26+ / macOS 26+ с поддерживаемыми устройствами.
- iCloud-синхронизация требует настроенных CloudKit entitlements и реального Apple ID.
- Биометрика (Face ID/Touch ID) тестируется на реальном устройстве; в симуляторе — через меню Features → Face ID.
- Виджеты (WidgetKit) тестируются отдельно в таргете FinikWidgets.
- Архивный экспорт/импорт `.finik` проверять с реальным файловым провайдером (iCloud Drive/Files), т.к. поведение picker может отличаться в симуляторе.
- Тесты с большими объёмами данных рекомендуется выполнять на реальном устройстве, а не в симуляторе.
